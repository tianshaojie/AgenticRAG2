# API Contract — Agentic RAG

## Base URL
```
/api/v1
```
OpenAPI 3.1 spec auto-generated by FastAPI at `/api/v1/openapi.json`.

## Endpoints

### Health

| Method | Path | Status | Description |
|--------|------|--------|-------------|
| GET | `/health` | ✅ Step 1 | Liveness check |
| GET | `/ready` | ✅ Step 1 | Readiness check (DB ping) |

#### GET /health
```json
{
  "status": "ok",
  "version": "0.1.0",
  "environment": "development"
}
```

#### GET /ready
```json
{
  "ready": true,
  "checks": {
    "database": true
  }
}
```

### Documents

| Method | Path | Status | Description |
|--------|------|--------|-------------|
| POST | `/documents` | 🔜 Step 2 | Upload and ingest document |
| GET | `/documents` | 🔜 Step 2 | List documents (paginated) |
| GET | `/documents/{id}` | 🔜 Step 2 | Get document detail |
| DELETE | `/documents/{id}` | 🔜 Step 2 | Soft-delete document |
| POST | `/documents/{id}/index` | 🔜 Step 3 | Trigger (re-)indexing |

#### POST /documents (Step 2)
- Content-Type: `multipart/form-data`
- Fields: `file` (required), `title` (optional), `meta` (optional JSON string)
- Response 202: `APIResponse<DocumentResponse>`

#### POST /documents/{id}/index (Step 3)
```json
{ "force_reindex": false }
```
Response 202: `APIResponse<DocumentIndexResponse>`

## Response Envelope

All endpoints return the standard envelope:
```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "request_id": "uuid"
}
```

Error responses:
```json
{
  "success": false,
  "data": null,
  "error": "Human-readable error message",
  "request_id": "uuid"
}
```

## Request Tracing Headers

| Header | Direction | Description |
|--------|-----------|-------------|
| `X-Request-ID` | Response | Server-generated UUID per request |
| `X-Trace-ID` | Request + Response | Forwarded or generated trace ID |

## Abstain Contract

When the agent has insufficient evidence:
```json
{
  "answer": "I cannot find sufficient evidence in the available documents to answer this question.",
  "abstained": true,
  "citations": []
}
```
Clients MUST check `abstained` flag before rendering the answer as authoritative.
